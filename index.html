HTML

  • Used to enter the content of the application: headers, lists, paragraphs
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • 'hX' = header 1, 2, etc.
  • 'li' = list
  • 'ol' = ordered list
  • 'ul' = unordered list
  • 'p' = paragraph

CSS

  • CSS defines the visuals of the application; colours, shapes, text justification, etc.
  • Can be embedded or usually have a separate linked file; linked in HTML file.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it

  • Sequence for git flow:
    1. Confirm branch: git status
    2. Navigate to feature branch: git checkout branch-name
    3. Add changed files from working area to staging area: git add -A
    4. Commit repository: git commit -m "change message"
    5. Ensure current branch is up to date: git pull origin main
    6. Push changes to repository: git push origin branch-name
    7. Create pull request > GitHub
    8. Merge request > GitHub (combines changes made in branch with main)
    9. Update local main branch: git checkout main > git pull origin main

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • if loops compares a variable with a condition
  • Right-click > Inspect to open browser-based console
